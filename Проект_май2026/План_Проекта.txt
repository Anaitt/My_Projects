Проект ИИ в работе сотрудников. ИИ360

**Задача:**
Выявить оптимальный уровень использования AI-инструментов для максимальной продуктивности и минимального риска выгорания, построив регрессионную модель и кластеризацию по датасету сотрудников.

**Ожидаемый Результат:** Оценка данных по корреляции, оценка R² для регрессии, опеределить количество кластеров сотрудников использующих ИИ связанных с факторами продуктивности сотрудников, таблица с топ-5 факторами влияния.

**План работ:** Применяемые решения для группы до 6, предполагается разбиться на 2 группы или одна группа с ролями
Группа 1:
- Подготовка данных и исследовательский анализ 
- Feature engineering: ai_ratio = ai_usage_hrs/workload_hrs
- График 1: корреляционной матрицы использования к общей нагрузке
- График 2: AI_usage vs productivity
- График 3: Boxplot workload по кластерам worklife_balance
- График 4: Heatmap корреляций (productivity с 7 features)
- Описательная статистика по группам
- T-test: productivity между high/low AI users (p-value)
- Автокорреляция stress_level по месяцам
Общий результат гипотезы и инсайты по обработанным данным

Группа 2
- Построение модели с использованием sklearn LinearRegression (по productivity + high_burnout)
- R², коэффициенты, feature importance (abs(coef))
- Residuals plot + Q-Q plot для валидности
- Применение методов кластеризации KMeans, Elbow + Silhouette для определения оптимального числа кластеров
- Профили кластеров: mean() по 9 метрикам
- 2D визуализация кластеров
- Оптимальная зона AI
- Финальный отчет: таблица инсайды + рекомендации и топ-5 факторами влияния

**Ценность:** ИИ в работе — ключевой тренд 2026, востребовано в аналитике по сотрудникам компаний для оптимизации внедрения ИИ в рабочие процессы и выработки стратегии внедрения ИИ.
